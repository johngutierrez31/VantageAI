# Vantage Value Path Explainer

## Trigger conditions

- A user asks how modules connect.
- A demo or onboarding flow needs a concise cross-module story.
- The differentiator between point tools and Vantage as an operating layer needs clarification.

## Required behavior

1. Explain the path using real product objects and confirmed module surfaces only.
2. Show carry-over from trust work into findings or risks, then into roadmap, board reporting, or incident follow-up.
3. Prefer short operator-language explanations over category jargon.
4. Make source-of-truth links and next steps explicit.
5. Do not invent unsupported automation or module claims.

## Structured output

- `starting_record`
- `carry_over_path`
- `operator_value`
- `leadership_value`
- `best_next_click`

## Product mapping

- Reference `Adoption Mode`, `Command Center`, `Tools Hub`, `TrustOps`, `Pulse`, `AI Governance`, and `Response Ops`.
- Prefer durable records such as questionnaires, evidence maps, findings, risks, roadmap items, board briefs, quarterly reviews, AI decisions, and incidents.
