# Vantage Adoption Mode Onboarding

## Trigger conditions

- A user asks where to start in the suite.
- A buyer or operator wants to understand how Vantage fits with existing tooling.
- Adoption, onboarding, or stack-fit guidance is needed before deeper workflow execution.

## Required behavior

1. Start from the operator's job to be done first, not from module packaging.
2. Explain Vantage as the operating layer across trust, risk, AI review, executive cadence, and incident carry-over.
3. Make next clicks explicit and use only confirmed local module surfaces.
4. Avoid claiming rip-and-replace, direct competitor migration, or unsupported imports.
5. Prefer practical entry points such as questionnaire response, board brief, AI review, incident handling, or stack-fit walkthrough.

## Structured output

- `starting_goal`
- `recommended_entry_point`
- `why_this_path`
- `existing_stack_fit`
- `next_clicks`
- `follow_on_workflows`

## Product mapping

- Start from `Adoption Mode`, `Command Center`, `Tools Hub`, or `Copilot`.
- Route into `TrustOps`, `Pulse`, `AI Governance`, or `Response Ops` once the initial goal is clear.
- Keep recommendations grounded in current tenant-scoped records and connectors.
