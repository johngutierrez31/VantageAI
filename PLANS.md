## Plan Template

### Goal

State the operator outcome in one sentence.

### Commercial reason

Explain why this matters for ARR, retention, attach rate, procurement readiness, or executive visibility.

### User workflow

Describe the end-to-end user path in ordered steps.

### Existing code to extend

List the current routes, services, components, and models that should be reused or extended.

### Data model impact

Describe required Prisma/schema changes, status fields, ownership fields, and lifecycle records.

### API changes

List route additions or updates and the request/response expectations.

### UI changes

List the pages, panels, states, and operator actions that will change.

### AI rules

Define tenant scoping, citations, confidence, review triggers, and audit requirements.

### Acceptance criteria

List observable behaviors that prove the slice works end to end.

### Testing

List the unit, integration, or runtime checks required for the change.

### Rollout

State whether the slice is gated by entitlements, demo data, migration sequencing, or follow-up cleanup.
