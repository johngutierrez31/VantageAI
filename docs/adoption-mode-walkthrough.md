# Adoption Mode Walkthrough

## Best quick path

1. Open `Command Center` and point out the cross-module operating view.
2. Click `Adoption Mode`.
3. Use the `Start Here` cards to frame the operator’s job to be done.
4. Show `Import Existing Work` and explain manual, CSV, and connector-assisted export modes.
5. Use the `Value Carry-Over` section to connect TrustOps, Pulse, and Response Ops.
6. Jump into the destination workflow from the chosen start path.

## Talk track

- Vantage does not require a rip-and-replace decision to become useful.
- Teams can keep Slack, Jira, Confluence, and spreadsheet exports while Vantage becomes the operating layer across trust, risk, leadership review, AI adoption, and incident carry-over.
- The first useful move is narrow: import only the records that improve the next urgent workflow.
- The value comes from carry-over, not from recreating every existing system inside Vantage.

## Suggested demo examples

- Questionnaire pressure:
  - Show TrustOps plus imported approved answers.
  - Explain how approved answers reduce rework before the next buyer request.
- Leadership reporting:
  - Show Pulse as the executive layer on top of imported or existing risks.
  - Follow the path into roadmap and board brief.
- Incident carry-over:
  - Show Response Ops and the path into findings and Pulse risk.

## Watchouts

- Do not imply direct competitor API imports if they are not implemented.
- Do not pitch Adoption Mode as a bulk migration tool.
- Keep the story practical: imports are narrow, durable, and operator-driven.
