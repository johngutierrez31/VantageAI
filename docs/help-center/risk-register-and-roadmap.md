# Risk Register and Roadmap

## Risk Register

### What it is for

The Risk Register is the durable catalog of business-relevant security risks.

### Sources of risks

Risks can come from:

- findings
- evidence gaps
- rejected trust answers
- assessments
- AI governance issues
- incidents and tabletops
- manual operator input

### How to use it

1. Review new or synced risks
2. Set severity, likelihood, and impact clearly
3. Assign an owner
4. Set target dates where appropriate
5. Use the filter views to manage open and overdue work

### Best use cases

- executive risk review
- remediation prioritization
- ownership tracking

## Roadmap

### What it is for

The Roadmap turns risk and remediation data into a 30/60/90 execution plan.

### How to use it

1. Generate roadmap items from top risks and open work
2. Review whether the plan is realistic
3. Assign owners and due dates
4. Track movement from planned to done
5. Use the roadmap in board and quarterly review discussions

### Best use cases

- leadership planning
- quarter planning
- communicating what security is doing next

## Operator tip

Do not keep the risk register abstract. If a risk has no owner, no due date, and no roadmap consequence, it is not operationally managed.
