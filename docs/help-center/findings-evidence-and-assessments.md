# Findings, Evidence, and Assessments

## Findings / Gaps

Use Findings / Gaps as the persistent backlog of security problems that came out of trust work, posture reviews, AI governance, and incidents.

Use it to:

- make weak support visible
- assign remediation ownership
- track cross-module issues in one place
- connect findings to risk and roadmap work

## Evidence

Use the Evidence area as the vault for reusable supporting material.

Keep it clean and useful:

- upload meaningful evidence
- prefer durable, reusable artifacts
- link evidence where it supports trust or governance claims
- refresh stale evidence before it becomes a liability

## Assessments

Use Assessments when you want structured readiness reviews and scoring.

Typical flow:

1. create or open the assessment
2. complete responses
3. score it
4. generate the report
5. turn important gaps into tasks or findings

## Best use cases

- internal readiness reviews
- recurring control checks
- preparing posture inputs for Pulse
- giving operators a structured place to review control coverage
