# Command Center

## What this page is for

Command Center is the operating center of the suite.

It is the best page to open first because it shows cross-module work in one place instead of making you inspect TrustOps, Pulse, AI Governance, and Response Ops separately.

## When to use it

Use Command Center when you need to answer any of these questions:

- What needs attention right now?
- What is overdue?
- What is affecting posture or buyer readiness?
- Where is executive pressure likely to come from next?
- Which workflow should I open next?

## What you should look at first

### Open work and overdue work

Start with the cards and backlog metrics. They tell you where security debt is building.

### Pulse executive layer

This tells you whether posture is improving, which top risks are open, and whether roadmap items are slipping.

### TrustOps operational pulse

This shows if trust work is backing up, if evidence is missing, or if review queues are becoming a bottleneck.

### AI Governance operational layer

Use this to see whether AI approvals are pending, blocked, or creating risk pressure.

### Response Ops operational layer

Use this to see whether incidents, post-incident actions, or tabletop work are affecting attention.

## How to use it well

1. Review the KPI cards
2. Open the module with the highest-pressure issue
3. Use the quick links to jump into the exact page you need
4. Return to Command Center after updates to confirm carry-over improved

## Best use cases

- daily security review
- weekly operating meeting prep
- leadership check-in prep
- deciding what to do before opening a deeper workflow

## What good looks like

A healthy Command Center has:

- low overdue review pressure
- low trust backlog
- a current Pulse snapshot
- owned risks with due dates
- no unmanaged incident work
- clear next-step actions
