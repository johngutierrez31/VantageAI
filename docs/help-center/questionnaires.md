# Questionnaires

## What this page is for

Questionnaires is the structured workspace for uploaded buyer question sets.

## What it does

- stores the intake record
- persists questionnaire rows and order
- normalizes question text
- maps rows to controls, evidence, and prior answers
- generates structured drafts
- forces review where confidence or support is weak
- exports approved answers only

## How to use it

1. Upload the questionnaire
2. Review the imported rows and preserved source order
3. Map rows to evidence and existing reusable answers
4. Generate answer drafts
5. Review row-level citations, confidence, and review state
6. Approve or reject each answer as needed
7. Export approved answers only

## What to pay attention to

### Confidence

Low-confidence answers should not be sent externally without human review.

### Evidence strength

If support is weak or missing, create follow-up work instead of bluffing.

### Reuse opportunity

If an answer is strong and repeatable, promote it to the Answer Library.

## Best use cases

- standard security questionnaires
- vendor diligence forms
- recurring enterprise procurement requests
- turning repeated buyer questions into reusable assets
