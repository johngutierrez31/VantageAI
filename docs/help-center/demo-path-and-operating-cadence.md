# Demo Path and Operating Cadence

## Best demo path

Use this sequence for a clean end-to-end walkthrough:

1. Command Center
2. TrustOps dashboard
3. Questionnaires
4. Trust packet or evidence map
5. Pulse dashboard
6. Risk register or board brief
7. AI Governance dashboard
8. AI use case or AI vendor intake
9. Response Ops dashboard
10. active incident or tabletop
11. back to Command Center

## Best artifacts to show

Show outputs that prove the suite is durable and operator-friendly:

- trust packet export
- approved answer library content
- Pulse board brief
- AI conditional approval
- active incident timeline
- approved after-action report

## Weekly operating cadence

### Daily

- Command Center
- urgent trust work
- incidents or active response tasks
- critical findings

### Weekly

- trust backlog review
- answer reuse review
- risk review
- roadmap review
- AI review queue
- weekly brief generation

### Monthly

- Pulse snapshot
- board brief refresh
- evidence freshness review
- policy review where needed

### Quarterly

- quarterly review
- roadmap reset and decision capture
- tabletop exercise
- leadership and board reporting

## Quickest wow-moment sequence

If you need a short demo, use:

1. Command Center
2. Trust packet
3. Pulse board brief
4. active incident
5. back to Command Center

That shows the suite as one connected operating system instead of four separate products.
