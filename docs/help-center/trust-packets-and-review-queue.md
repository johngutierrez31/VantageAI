# Trust Packets and Review Queue

## Trust Packets

### What this page is for

Trust Packets assemble buyer-ready packages from approved materials only.

### Use it when

- you want a clean external package instead of a raw questionnaire file
- a buyer wants a summary set of trust materials
- you need repeatable external-share artifacts

### How to use it

1. Choose internal review mode or external share mode
2. Assemble the relevant sections
3. Review stale or expiring evidence indicators
4. Verify no internal-only notes are included
5. Export the packet in a share-safe format

### Best use cases

- enterprise sales acceleration
- customer renewal trust refresh
- repeat buyer package reuse

## TrustOps Review Queue

### What this page is for

The review queue is the operating surface for assignment, due dates, overdue visibility, and review decisions across TrustOps objects.

### Use it when

- drafts are waiting for approval
- evidence maps need a reviewer
- a trust packet is close to a customer deadline
- you need to rebalance reviewer workload

### How to use it

1. Filter by status, reviewer, overdue, or object type
2. Assign the correct reviewer
3. Set or adjust due dates
4. Review notes and approval history
5. Approve, reject, or reassign work

### Best use cases

- keeping trust work moving under deadline pressure
- managing part-time reviewers
- making overdue buyer-readiness work visible
