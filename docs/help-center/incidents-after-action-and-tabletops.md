# Incidents, After-Action Reports, and Tabletop Exercises

## Incidents

### What the incident record is for

The incident record is the main operational object for live response.

### How to use it

1. Create the incident manually or from a scenario type
2. Set severity, owner, and status
3. Capture affected systems, services, or vendors
4. Launch the runbook task pack
5. Add timeline and decision events as the incident evolves
6. Close into post-incident review

### Best use cases

- phishing or identity compromise
- vendor breach notice
- ransomware or containment work
- cloud exposure
- AI misuse incident if AI governance is relevant

## After-Action Reports

### What they are for

After-action reports formalize what happened, what was done, what was learned, and what still needs action.

### How to use them

1. Generate the report from the incident and timeline
2. Review the summary and lessons learned
3. Approve before export
4. Convert follow-up actions into tracked work

### Best use cases

- executive post-incident review
- customer-facing incident explanation preparation
- internal remediation planning

## Tabletop Exercises

### What they are for

Tabletops provide lightweight exercise planning and follow-up capture.

### How to use them

1. Create the scenario
2. Set participants and date
3. Capture decisions made during the exercise
4. Record gaps identified
5. Convert gaps into findings, risks, and tasks

### Best use cases

- ransomware rehearsal
- third-party breach rehearsal
- identity compromise rehearsal
- AI misuse rehearsal
