# Getting Started

## What VantageAI is for

VantageAI is the security operating system for lean teams under procurement pressure.

It is designed to help a small security function do four connected jobs:

- Prove trust and respond to buyer security diligence
- Give leadership one cyber posture view with clear ownership and remediation
- Govern AI usage safely and operationally
- Improve first-hour security operations and response readiness

## How to use VantageAI the way it was designed

Use the suite with this operating model in mind.

### Mission

Build VantageAI as the security operating system for lean teams under procurement pressure.

This means the app should be used as the system of record for trust work, posture management, AI governance, and incident operations. It is not meant to be a disconnected collection of admin screens.

### Commercial priorities reflected in the product

The suite is organized in this order:

1. TrustOps
2. Pulse
3. AI Governance
4. Response Ops

That order matters when deciding where to start.

- Start with TrustOps if customer trust, diligence, and evidence pressure is immediate
- Use Pulse to turn operational work into leadership-readable posture and risk
- Use AI Governance to keep AI adoption governed and auditable
- Use Response Ops to handle incidents and exercises with durable records

### Product surfaces to use first

If you are not sure where to start, open these surfaces first:

- Command Center
- Trust Inbox
- Findings / Gaps
- Policies
- Copilot
- Runbooks
- Security Analyst
- Cyber Range
- Questionnaires
- Assessments

### What the system expects from operators

The app works best when you treat outputs as durable records, not temporary notes.

- Use tenant-scoped records for meaningful work
- Reuse evidence and approved answers instead of rewriting material every time
- Review and approve high-stakes outputs before sharing them externally
- Turn weak support, missing evidence, rejected items, and incident lessons into tracked findings, risks, tasks, or roadmap work
- Use Pulse and board reporting to create recurring executive value, not just one-time deliverables

### What not to use it for

Do not treat VantageAI as:

- a generic compliance clone
- a blank AI chat surface
- a consumer security tool
- a side system for notes that never turn into tracked work

## Your best first week

### Day 1

- Open Command Center
- Review the module entry points
- Confirm users, tenant, and billing state
- Review seeded or existing records so you understand the data model

### Day 2

- Upload or open a questionnaire
- Review how drafts, evidence, answer reuse, and review states work
- Generate one trust packet

### Day 3

- Generate a Pulse snapshot
- Review open risks and overdue work
- Open the roadmap and board brief flow

### Day 4

- Register one AI use case and one AI vendor review
- Review policy conditions, blockers, and follow-up actions

### Day 5

- Open Response Ops
- Walk through one incident and one tabletop
- Review how after-action work flows back into findings, tasks, and risks

## If you are a solo operator

Use this sequence every week:

1. Command Center for priorities
2. TrustOps for buyer pressure
3. Findings / Gaps for persistent problems
4. Pulse for executive reporting and roadmap
5. AI Governance for new AI adoption requests
6. Response Ops for incident readiness and incident closeout

That is the intended use pattern.
