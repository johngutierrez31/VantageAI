# Trust Inbox

## What this page is for

Trust Inbox is the intake layer for buyer trust requests, diligence asks, and response work.

## Use it when

- you receive a new customer or prospect diligence request
- you want a durable record of a trust response cycle
- you need to coordinate attachments, questionnaires, and outbound deliverables

## How to use it

1. Create the trust inbox item
2. Record who the requester is and what they need
3. Link the questionnaire or supporting materials
4. Use the related pages to draft answers and build the packet
5. Export the approved response pack when ready

## Best use cases

- enterprise sales diligence
- renewal security reviews
- customer evidence requests
- packaging trust responses for repeated use

## Operator tip

Use Trust Inbox as the business-facing container for trust work. Use Questionnaires, Evidence Maps, and Trust Packets as the working components underneath it.
